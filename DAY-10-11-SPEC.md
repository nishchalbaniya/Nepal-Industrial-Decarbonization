# Day 10 + Day 11: HTTP API + Turnkey Installer

## Day 10 -- FastAPI server

The `nepal_decarb_pro.server` module wraps the cooler, kiln, calibration
and STEP / P&ID export in a single FastAPI app:

| Method | Path | Body | Returns |
| --- | --- | --- | --- |
| GET  | `/`                     | --                       | HTML dashboard (vanilla JS, no framework) |
| GET  | `/docs`                 | --                       | Swagger UI (auto-generated by FastAPI)     |
| GET  | `/api/version`          | --                       | module versions                            |
| GET  | `/api/status`           | --                       | tool stack (Python, NumPy, FreeCAD, ...)   |
| GET  | `/api/plants`           | --                       | available plant presets                    |
| POST | `/api/cooler/run`       | `{plant, overrides?}`    | cooler KPIs + written JSON path            |
| POST | `/api/kiln/run`         | `{plant, fuel?}`         | kiln KPIs                                  |
| POST | `/api/cooler/calibrate` | `{target, v04?, out?}`   | loss, RMSE, posterior, ship-gate bands     |
| POST | `/api/cooler/export-step` | `{output?}`            | v0.5.0 cooler STEP file path + size        |
| POST | `/api/kiln/export-step` | `{output?}`              | PlantA kiln STEP file path + size         |
| POST | `/api/cooler/export-pid` | `{output?}`             | P&ID STEP+SVG+JSON outputs                 |
| GET  | `/api/pid-svg`          | --                       | inline SVG of the cooler P&ID              |
| GET  | `/api/artifacts`        | --                       | list files in `pro/demo-output/`           |
| GET  | `/api/artifacts/{name}` | --                       | download a file                            |

The server binds to `127.0.0.1` (localhost only) on port 8000 by default.
No data leaves the machine. The dashboard is a single HTML file with
inline CSS + vanilla JS so it works in IE/Edge/Chrome/Firefox without
any JS framework.

Run:
```
python -m nepal_decarb_pro.cli serve --port 8000
```
Or:
```
nepal-decarb serve --port 8000
```

## Day 11 -- turnkey installer

The `nepal-decarb setup` subcommand creates a complete set of
shortcuts so the user can launch the dashboard with a double-click.

What it does:
1. Calls `nepal-decarb install` to write `nepal-decarb.bat` +
   `nepal-decarb.ps1` to `pro/dist/`.
2. Copies those into `Desktop\NepalDecarb\` and
   `Start Menu\Programs\NepalDecarb\`.
3. Writes `Start NepalDecarb Dashboard.vbs` to both:
   - sets `NEPAL_DECARB_ROOT` env var so the .bat can find the repo
   - launches `nepal-decarb serve --port 8000` in a hidden window
   - polls `http://127.0.0.1:8000/api/version` for up to 10s
   - opens the user's default browser to `http://127.0.0.1:8000/`
4. Writes `Run Demo (PlantA).bat` (one-shot end-to-end demo +
   opens the result folder).
5. Writes `Uninstall NepalDecarb.bat` (removes the Desktop +
   Start Menu entries; the repo and Python packages are not
   removed).
6. Writes `README.txt` with usage instructions.

Run:
```
nepal-decarb setup --port 8000
```

The `.bat` launcher has two repo-discovery modes:
1. If `NEPAL_DECARB_ROOT` is set, use it directly.
2. Otherwise, walk up from the .bat's location looking for
   `pro/src/nepal_decarb_pro/__init__.py`. This is what
   `pro/dist/nepal-decarb.bat` does; the Desktop copy uses mode 1
   because the VBS sets the env var.

## Why FastAPI

- Tiny dependency footprint (FastAPI + Uvicorn + httpx; already
  required for the test client).
- Auto-generated `/docs` (Swagger UI) so the user can browse
  endpoints without me writing documentation.
- Pydantic request models give us input validation for free.
- The server is local-only (`127.0.0.1`); no firewall punch-through,
  no public URL, no data leaving the machine.

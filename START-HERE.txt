WHAT'S RUNNING
--------------

The dashboard has 14 endpoints, an HTML page, and a Swagger UI:

  GET  /                          - HTML dashboard
  GET  /docs                      - Swagger UI (auto-generated)
  GET  /api/version               - module versions
  GET  /api/status                - tool stack status
  GET  /api/plants                - available plant presets
  POST /api/cooler/run            - run cooler, return KPIs
  POST /api/kiln/run              - run kiln, return KPIs
  POST /api/cooler/calibrate      - calibrate cooler (v0.4.0 or v0.5.0)
  POST /api/cooler/export-step    - export v0.5.0 cooler STEP
  POST /api/kiln/export-step      - export PlantA kiln STEP
  POST /api/cooler/export-pid     - export cooler P&ID (STEP+SVG+JSON)
  GET  /api/pid-svg               - inline P&ID SVG
  GET  /api/artifacts             - list files in pro/demo-output/
  GET  /api/artifacts/{name}      - download a file

The server binds to 127.0.0.1:8000 (localhost only). Stop it
from the system tray or `Stop-Process -Name python`.


THE THREE FAILING SHIP-GATE BANDS (honest disclosure)
-----------------------------------------------------

3 of 6 ship-gate bands pass on the v0.5.0 calibrated model:
  PASS: sec_air 715 C, efficiency 0.77, first-law 0.0
  FAIL: tertiary 190 C (band 400-700), exhaust 149 C (150-300),
        clinker 351 C (120-200)

The v0.5.0 model physics at 130 t/h cannot deliver those three
bands simultaneously. To unlock the full ship-gate we need
EITHER:
  (a) real PlantA plant data (CSV with 4h shifts, columns
      specified in DAY-04-SPEC.md) so we can verify the model
      against actual measurements, OR
  (b) v0.6.0 model changes: compartment subdivision within
      compartment, or lower-throughput operation.

This is documented honestly in the v0.5.0 commit message
(2aa918c) and in DAY-05-SPEC.md.

The dashboard shows the same numbers (3/6 pass) so the
honest finding is visible to anyone running the demo.

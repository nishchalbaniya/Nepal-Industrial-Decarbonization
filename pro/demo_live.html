🚀 § How to deploy this for all of Nepal's cement industry

Three paths. Pick the one that matches your scale. Start with #1, scale to #2 when needed, add #3 only when users want the LLM advisor.

Hetzner / DigitalOcean VPS (5 min to live)

Recommended for: pilot, NCMA cohort, plants 1-20. Costs €8.50/mo. Deploy from a fresh Ubuntu 22.04 in Mumbai region (closest to Nepal).

  1. Rent Hetzner CX21 (Mumbai region, €4.85/mo for ARM, €8.50/mo for x86) at hetzner.com/cloud
  2. Point your domain (e.g. nepalcarbon.org.np) at the server IP via Cloudflare DNS
  3. SSH in and run the one-command deploy:

    DOMAIN=carbon.example.com ADMIN_EMAIL=ops@example.com \
      curl -sSL https://raw.githubusercontent.com/.../deploy.sh | sudo bash
  4. Wait 5 min. The script: installs Docker, sets up Postgres + FastAPI + Streamlit + MQTT + Redis, configures nginx + Let's Encrypt, sets daily backups, enables UFW firewall
  5. Run the pilot smoke test:
    cd /opt/nepal_decarb_pro && python3 scripts/run_pilot.py
  6. Verify everything is live:
    python3 scripts/verify_deployment.py http://carbon.example.com
  7. Access the platform:
    https://carbon.example.com/ (Streamlit)
    https://carbon.example.com/docs (FastAPI)
    https://carbon.example.com:3000 (Grafana)

AWS production (Terraform, ~30 min to live)

Recommended for: 20+ plants, national rollout, 99.9% uptime SLA. Provisions ECS Fargate, RDS multi-AZ, ALB, S3, KMS, ACM, IAM, CloudWatch.

  1. Get an AWS account + IAM user with admin
  2. Install Terraform ≥1.5
  3. Configure the deploy:
    cd deploy/terraform
    export TF_VAR_db_password=$(openssl rand -base64 24)
    export TF_VAR_jwt_secret=$(openssl rand -base64 48)
    export TF_VAR_mqtt_password=$(openssl rand -base64 16)
    export TF_VAR_domain_name=carbon.example.com
  4. Apply:
    terraform init && terraform apply (15-20 min)
  5. Build and push Docker images:
    aws ecr get-login-password ... | docker login ...
    docker build -t $ECR_API:v1.1.1 . && docker push $ECR_API:v1.1.1
  6. Point Route53 at the ALB
  7. Verify:
    python3 scripts/verify_deployment.py https://carbon.example.com

GPU + vLLM (for the LLM advisor, optional)

Recommended for: 50+ active users asking the advisor in Nepali. The CPU stub works fine for pilots. Don't add GPU until you have real users asking questions.

  1. Rent a GPU instance — Lambda Labs A10 ($0.60/hr) is the sweet spot
  2. Install NVIDIA Container Toolkit (one-time, 5 min)
  3. Start vLLM:
    cd deploy/vllm && docker compose up -d
  4. Wait 5 min for Qwen2.5-7B-AWQ to load. Check:
    curl http://localhost:8001/v1/models
  5. On the API host, point at the GPU:
    export LLM_ENDPOINT=http://gpu-host:8001
    export LLM_MODEL=Qwen/Qwen2.5-7B-Instruct-AWQ
    export LLM_BACKEND=vllm
    docker restart nepal-decarb-api
  6. Test:
    curl -X POST http://api:8000/api/v1/advisor/ask -d '{...}'
  7. Cost tip: $30/hr GPU cap is overkill. For advisor Q&A, A10 at $0.60/hr on-demand 8 hr/day = $144/mo. For fine-tuning, burst the A100 for 3 hr one-time = $3.

📋 § First 24 hours after going live

  1. Run the pilot to confirm the platform works (5 min)
  2. Run the verification script to confirm all 11 endpoints are ✅ (1 min)
  3. Onboard Hetauda Cement as your reference plant (Express, 15 min)
  4. Generate the 5 reference reports (Executive, ISO 14064-1, Verra, TCFD, SBTi) for Hetauda (2 min)
  5. Onboard 2-3 more plants from the NCMA list (Express each, 15 min each)
  6. Set up Cloudflare in front of the domain (free tier, DDoS + caching)
  7. Set up SMTP for alert emails (Gmail SMTP, free)
  8. Smoke-test the GPU path on-demand (don't keep it running 24/7)
  9. Write a 1-page ops runbook for whoever's on call

❌ Don't do these in the first 24 hours

  • Don't keep the $30/hr GPU running 24/7 — you'll burn $720/day for zero user benefit
  • Don't skip the pilot smoke test — it's the only way to confirm every module works
  • Don't onboard plants before verifying the platform yourself
  • Don't deploy to AWS for a pilot — start with the $20/mo VPS
  • Don't add custom features before the basic platform is live

Operations runbook

Health checks and common fixes.

curl -fsS http://localhost:8000/health && echo "API OK"

Common issues

  • API down: docker logs --tail 50 nepal-decarb-api · docker restart nepal-decarb-api
  • DB full: prune old sensor data · VACUUM FULL
  • MQTT sensors offline: check ACL, password, plant's internet
  • LLM advisor 500: check docker logs nepal-decarb-vllm · falls back to stub
  • SSL expired: certbot renew && systemctl reload nginx
  • Tests fail: pip install -U pydantic

Onboard a new plant

7 steps. Express = 15 min. Full = 2-4 hr with historical data.
  1. Create tenant — sign-up link, pick email + language
  2. Register plant — name, type, location, capacity, fuel
  3. Add monthly data — CSV bulk import for historical, web form for ongoing
  4. Compute baseline — Tier 2 (or Tier 3 with TOC), 5,000-sample Monte Carlo
  5. Set up IoT (optional) — ESP32 + DHT22/MQ-7/MQ-135/MAX31855, MQTT
  6. Configure alerts — kiln T, CO, NOx, production drop
  7. Generate reports — Executive, ISO 14064-1, Verra, TCFD, SBTi (all PDFs)
